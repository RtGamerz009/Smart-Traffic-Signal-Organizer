====================================================
🎯 PROJECT GOAL
====================================================
Create a modern, interactive, simulation-based web application that solves this interview challenge:

“Design a traffic signal system for a 4-road intersection where each road has a different number of vehicles. Instead of using a fixed timer, dynamically decide which road gets the green signal and for how long, such that total waiting time across all roads is minimized while ensuring fairness.”

This should feel like a real-world intelligent traffic control product — not just a coding answer.

====================================================
🧠 CORE INTERVIEW LOGIC TO IMPLEMENT
====================================================
The system should model a smart adaptive traffic light controller using a:

✅ Priority-Based Dynamic Scheduling Algorithm

Use this scoring formula for each road:

Priority Score = 
(α × Queue Length) + 
(β × Waiting Time) + 
(γ × Arrival Rate) + 
(δ × Emergency Vehicle Flag)

Where:
- Queue Length = number of vehicles currently waiting
- Waiting Time = how long the road has been waiting
- Arrival Rate = how quickly new vehicles are arriving
- Emergency Vehicle Flag = if ambulance/fire/police is present, give maximum priority

The system must:
1. Continuously calculate priority scores for all 4 roads
2. Select the road with the highest score
3. Dynamically assign green light duration based on vehicle count and flow
4. Update traffic queues after each cycle
5. Prevent starvation (no road should wait forever)
6. Simulate real-time traffic flow intelligently

====================================================
⚙️ GREEN LIGHT DURATION LOGIC
====================================================
Green light time should NOT be fixed.

Implement dynamic green duration using:

Green Time = min(MaxGreenTime, max(MinGreenTime, QueueLength × TimePerVehicle))

Suggested defaults:
- MinGreenTime = 10 seconds
- MaxGreenTime = 60 seconds
- TimePerVehicle = 2 seconds

This means:
- More vehicles = longer green time
- Fewer vehicles = shorter green time
- No road gets unnecessary extra time

====================================================
🛡️ FAIRNESS / STARVATION PREVENTION
====================================================
Add fairness logic so one road is never ignored forever.

Use “Aging”:
- Every cycle a road is not selected, its Waiting Time score increases
- This gradually boosts its priority over time

This is critical and must be shown visually in the app.

====================================================
🚑 EMERGENCY VEHICLE OVERRIDE
====================================================
Add emergency handling:
- If a road has an ambulance / fire truck / police vehicle, it should get immediate top priority
- Override normal scheduling temporarily
- Show this visually in the interface
